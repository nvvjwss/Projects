Installation

การติดตั้งโดยใช้ Isaac Sim Pip Package

ขั้นตอนต่อไปนี้จะติดตั้ง Isaac Sim จาก pip ก่อน แล้วจึงติดตั้ง Isaac Lab จาก source code

⚠️
ข้อควรระวัง

การติดตั้ง Isaac Sim ด้วย pip ต้องการความเข้ากันได้กับ GLIBC เวอร์ชัน 2.35 ขึ้นไป เพื่อตรวจสอบเวอร์ชัน GLIBC บนระบบของคุณ ให้ใช้คำสั่ง ldd --version

ซึ่งอาจทำให้เกิดปัญหาความเข้ากันได้กับ Linux distributions บางตัว ตัวอย่างเช่น Ubuntu 20.04 LTS มี GLIBC 2.31 เป็นค่าเริ่มต้น หากพบปัญหาความเข้ากันได้ เราแนะนำให้ทำตามวิธีการ Isaac Sim Binaries Installation แทน

ℹ️
หมายเหตุ

หากคุณวางแผนที่จะตั้งค่า Visual Studio Code ในภายหลัง เราแนะนำให้ทำตามวิธีการ Isaac Sim Binaries Installation แทน

Step 1

การติดตั้ง Isaac Sim

ตั้งแต่ Isaac Sim 4.0 เป็นต้นไป สามารถติดตั้ง Isaac Sim โดยใช้ pip ได้ วิธีนี้ทำให้ติดตั้ง Isaac Sim ได้ง่ายขึ้นโดยไม่จำเป็นต้องดาวน์โหลด Isaac Sim binaries หากพบปัญหาใดๆ กรุณารายงานไปที่ Isaac Sim Forums

⚠️
ข้อควรระวัง

บน Windows อาจจำเป็นต้องเปิดใช้งาน long path support เพื่อหลีกเลี่ยงข้อผิดพลาดในการติดตั้งที่เกิดจากข้อจำกัดของ OS

การเตรียม Python Environment

การสร้าง Python environment เฉพาะเป็นสิ่งที่แนะนำอย่างยิ่ง ซึ่งช่วยให้:

  • หลีกเลี่ยงความขัดแย้งกับ system Python หรือโปรเจกต์อื่นๆ ที่ติดตั้งบนเครื่อง
  • แยก dependencies ออกจากกัน เพื่อให้การอัปเกรด packages หรือการทดลองในโปรเจกต์อื่นๆ ไม่ทำให้ Isaac Sim เสียหาย
  • จัดการ environments หลายตัวสำหรับการตั้งค่าที่มี dependencies เวอร์ชันต่างกันได้ง่ายขึ้น
  • ทำให้ reproducibility ง่ายขึ้น — environment มีเฉพาะ packages ที่จำเป็นสำหรับโปรเจกต์ปัจจุบัน ทำให้แชร์การตั้งค่ากับเพื่อนร่วมงานหรือรันบนเครื่องอื่นได้ง่ายขึ้น

คุณสามารถเลือก package manager ต่างๆ เพื่อสร้าง virtual environment:

  • UV: package manager สำหรับ Python ที่ทันสมัย รวดเร็ว และปลอดภัย
  • Conda: package manager สำหรับ Python ที่รองรับหลาย platform และหลายภาษา
  • venv: standard library สำหรับสร้าง virtual environments ใน Python
⚠️
ข้อควรระวัง — Python Version

Python version ของ virtual environment ต้องตรงกับ Python version ของ Isaac Sim:

  • สำหรับ Isaac Sim 5.X ต้องการ Python เวอร์ชัน 3.11
  • สำหรับ Isaac Sim 4.X ต้องการ Python เวอร์ชัน 3.10

การใช้ Python เวอร์ชันที่แตกต่างออกไปจะทำให้เกิดข้อผิดพลาดเมื่อรัน Isaac Lab

คำแนะนำต่อไปนี้สำหรับ Isaac Sim 5.X ซึ่งต้องการ Python 3.11 หากต้องการติดตั้ง Isaac Sim 4.5 กรุณาปรับแก้คำแนะนำให้ใช้ Python 3.10 แทน

สร้าง virtual environment โดยใช้ package manager ตัวใดตัวหนึ่ง:

Conda Environment

เพื่อติดตั้ง conda กรุณาทำตามคำแนะนำที่นี่ คุณสามารถสร้าง Isaac Lab environment โดยใช้คำสั่งต่อไปนี้ เราแนะนำให้ใช้ Miniconda เนื่องจากเป็นระบบจัดการ environment ที่เบาและประหยัดทรัพยากร

bash — Conda 
$ conda create -n env_isaaclab python=3.11
$ conda activate env_isaaclab

venv Environment

bash — venv (Linux) 
$ python3.11 -m venv env_isaaclab
$ source env_isaaclab/bin/activate

UV Environment (experimental)

bash — UV (Linux) 
$ uv venv env_isaaclab --python 3.11
$ source env_isaaclab/bin/activate

ตรวจสอบให้แน่ใจว่ามี pip เวอร์ชันล่าสุดติดตั้งอยู่ หากต้องการอัปเดต pip ให้รันคำสั่งต่อไปนี้จากภายใน virtual environment:

bash — Linux 
$ pip install --upgrade pip

การติดตั้ง Dependencies

ℹ️
หมายเหตุ

ในกรณีที่ใช้ UV เพื่อสร้าง virtual environment กรุณาแทน pip ด้วย uv pip ในคำสั่งต่อไปนี้

ติดตั้ง Isaac Sim pip packages:

bash 
$ pip install "isaacsim[all,extscache]==5.1.0" --extra-index-url https://pypi.nvidia.com

ติดตั้ง CUDA-enabled PyTorch build ที่ตรงกับ system architecture ของคุณ:

bash — Linux (x86_64) 
$ pip install -U torch==2.7.0 torchvision==0.22.0 --index-url https://download.pytorch.org/whl/cu128
bash — Windows (x86_64) 
> pip install -U torch==2.7.0 torchvision==0.22.0 --index-url https://download.pytorch.org/whl/cu128
bash — Linux (aarch64) 
$ pip install -U torch==2.7.0 torchvision==0.22.0

การตรวจสอบการติดตั้ง Isaac Sim

  1. ตรวจสอบให้แน่ใจว่า virtual environment ของคุณถูก activate แล้ว (ถ้ามี)
  2. ตรวจสอบว่า simulator ทำงานตามที่คาดไว้:
bash 
# หมายเหตุ: สามารถส่ง argument "--help" เพื่อดู arguments ทั้งหมดที่เป็นไปได้
$ isaacsim

นอกจากนี้ยังสามารถรันด้วย experience file เฉพาะได้:

bash 
# experience files สามารถเป็น absolute path หรือ relative path ที่ค้นหาใน isaacsim/apps หรือ omni/apps
$ isaacsim isaacsim.exp.full.kit
ℹ️
หมายเหตุ

เมื่อรัน Isaac Sim เป็นครั้งแรก extensions ที่ต้องพึ่งพาทั้งหมดจะถูกดึงมาจาก registry กระบวนการนี้อาจใช้เวลามากกว่า 10 นาที และจำเป็นต้องทำในการรันครั้งแรกของแต่ละ experience file เมื่อ extensions ถูก pull มาแล้ว การรันครั้งต่อๆ ไปโดยใช้ experience file เดียวกันจะใช้ extensions ที่ cache ไว้

⚠️
ข้อควรระวัง — การยอมรับ EULA

การรันครั้งแรกจะแจ้งให้ผู้ใช้ยอมรับ Nvidia Omniverse License Agreement ในการยอมรับ EULA ให้ตอบ Yes เมื่อถูกแจ้งด้วยข้อความด้านล่าง:

bash 
By installing or using Isaac Sim, I agree to the terms of NVIDIA OMNIVERSE LICENSE AGREEMENT (EULA)
in https://docs.isaacsim.omniverse.nvidia.com/latest/common/NVIDIA_Omniverse_License_Agreement.html
Do you accept the EULA? (Yes/No): Yes

หาก simulator ไม่ทำงานหรือ crash ขณะทำตามคำแนะนำข้างต้น แสดงว่ามีการกำหนดค่าบางอย่างที่ไม่ถูกต้อง เพื่อ debug และแก้ไขปัญหา กรุณาตรวจสอบ Isaac Sim documentation และ Isaac Sim Forums

Step 2

การติดตั้ง Isaac Lab

การ Clone Isaac Lab

ℹ️
หมายเหตุ

เราแนะนำให้ fork Isaac Lab repository เพื่อ contribute ให้กับโปรเจกต์ แต่ไม่จำเป็นสำหรับการใช้ framework หากคุณ fork กรุณาแทน isaac-sim ด้วย username ของคุณในคำแนะนำต่อไปนี้

Clone Isaac Lab repository เข้าไปใน workspace ของโปรเจกต์:

bash — SSH 
$ git clone https://github.com/isaac-sim/IsaacLab.git

เราให้ helper executable isaaclab.sh และ isaaclab.bat สำหรับ Linux และ Windows ตามลำดับ ซึ่งให้ utilities ในการจัดการ extensions

bash — Linux 
$ ./isaaclab.sh --help

usage: isaaclab.sh [-h] [-i] [-f] [-p] [-s] [-t] [-o] [-v] [-d] [-n] [-c] -- Utility to manage Isaac Lab.

optional arguments:
   -h, --help           Display the help content.
   -i, --install [LIB]  Install the extensions inside Isaac Lab and learning frameworks (rl_games, rsl_rl, sb3, skrl) as extra dependencies. Default is 'all'.
   -f, --format         Run pre-commit to format the code and check lints.
   -p, --python         Run the python executable provided by Isaac Sim or virtual environment (if active).
   -s, --sim            Run the simulator executable (isaac-sim.sh) provided by Isaac Sim.
   -t, --test           Run all python pytest tests.
   -o, --docker         Run the docker container helper script (docker/container.sh).
   -v, --vscode         Generate the VSCode settings file from template.
   -d, --docs           Build the documentation from source using sphinx.
   -n, --new            Create a new external project or internal task from template.
   -c, --conda [NAME]   Create the conda environment for Isaac Lab. Default name is 'env_isaaclab'.
   -u, --uv [NAME]      Create the uv environment for Isaac Lab. Default name is 'env_isaaclab'.

การติดตั้ง

ติดตั้ง dependencies โดยใช้ apt (บน Linux เท่านั้น):

bash — Linux 
# dependency เหล่านี้จำเป็นสำหรับ robomimic ซึ่งไม่มีให้ใช้งานบน Windows
$ sudo apt install cmake build-essential

รัน install command ที่จะวนซ้ำ extensions ทั้งหมดใน source directory และติดตั้งโดยใช้ pip (พร้อม --editable flag):

bash — Linux 
$ ./isaaclab.sh --install # หรือ "./isaaclab.sh -i"

โดยค่าเริ่มต้น คำสั่งข้างต้นจะติดตั้ง learning frameworks ทั้งหมด ซึ่งรวมถึง rl_games, rsl_rl, sb3, skrl, robomimic

หากต้องการติดตั้งเฉพาะ framework เฉพาะ สามารถส่งชื่อ framework เป็น argument ได้ ตัวอย่างเช่น หากต้องการติดตั้งเฉพาะ rl_games framework:

bash — Linux 
$ ./isaaclab.sh --install rl_games  # หรือ "./isaaclab.sh -i rl_games"

ตัวเลือกที่ถูกต้องได้แก่ all, rl_games, rsl_rl, sb3, skrl, robomimic และ none หาก none ถูกส่ง จะไม่มี learning frameworks ถูกติดตั้ง

การตรวจสอบการติดตั้ง Isaac Lab

เพื่อยืนยันว่าการติดตั้งสำเร็จ ให้รันคำสั่งต่อไปนี้จาก root ของ repository:

bash — Linux 
# ตัวเลือกที่ 1: ใช้ isaaclab.sh executable
# หมายเหตุ: ใช้ได้กับทั้ง bundled python และ virtual environment
$ ./isaaclab.sh -p scripts/tutorials/00_sim/create_empty.py

# ตัวเลือกที่ 2: ใช้ python ใน virtual environment
$ python scripts/tutorials/00_sim/create_empty.py

คำสั่งข้างต้นควรเปิด simulator และแสดงหน้าต่างที่มี black viewport คุณสามารถออกจาก script ได้โดยกด Ctrl+C บน terminal บนเครื่อง Windows กรุณาหยุด process จาก Command Prompt โดยใช้ Ctrl+Break หรือ Ctrl+fn+B

✅
การติดตั้งสำเร็จ!

หากเห็นหน้าต่าง simulator พร้อม black viewport แสดงว่าการติดตั้งสำเร็จแล้ว

ℹ️
หมายเหตุ

หากพบ error ModuleNotFoundError: No module named 'isaacsim' กรุณาตรวจสอบให้แน่ใจว่า virtual environment ถูก activate และได้รัน source _isaac_sim/setup_conda_env.sh แล้ว (สำหรับ uv ด้วย)
